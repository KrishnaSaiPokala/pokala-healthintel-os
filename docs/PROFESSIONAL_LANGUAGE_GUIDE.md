# Professional Language Guide

This project should read as a serious healthcare IT and model-evaluation artifact.

## Preferred public language

- Public healthcare intelligence workspace
- Healthcare IT intelligence
- Evidence-linked market diligence
- Claim-boundary governance
- Public-source data lineage
- Transparent model benchmarking
- Research benchmark
- Model evaluation workspace
- No PHI
- Not clinical decision support
- No patient-level prediction

## Avoid in public-facing copy

- Internal release-proof labels
- Internal milestone names
- Startup-style launch labels
- Inflated superlatives
- Hype claims
- Any language implying validated forecasting
- Any language implying clinical decision support
- Any language implying patient-level prediction

## Positioning standard

The project should show healthcare data judgment, evidence governance, model transparency, and deployment discipline without overclaiming.
